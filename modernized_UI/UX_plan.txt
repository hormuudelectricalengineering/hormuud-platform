 UI/UX Modernization Plan

     Context

     The Hormuud Electrical Services app has a functional but dated UI. The current design uses basic components with minimal    
     styling, flat colors, and lacks visual hierarchy. To provide a professional, polished, and minimalist experience that
     matches modern standards, we need to implement a comprehensive UI/UX overhaul.

     Goals

     - Modern, minimalist aesthetic with clean lines and ample whitespace
     - Consistent design system across all screens
     - Improved visual hierarchy and user guidance
     - Enhanced user experience with subtle animations and transitions
     - Professional color palette with better contrast and depth
     - Polished micro-interactions and feedback

     Current State Analysis

     - Color Scheme: Basic primary color (#001F5C) with flat backgrounds
     - Typography: Standard system fonts with limited hierarchy
     - Components: Basic cards, buttons, inputs with minimal styling
     - Spacing: Inconsistent padding/margins
     - Animations: None present
     - Visual Depth: Flat design with basic shadows

     Proposed Design System

     1. Color Palette

     Primary: Deep Navy (#0A2463) - more sophisticated than current #001F5C
     Secondary: Accent Blue (#0066CC) for interactive elements
     Background: Off-white (#F9FAFB) for secondary backgrounds
     Surface: White (#FFFFFF) for cards
     Text: Dark Gray (#1F2937) for headings, Medium Gray (#6B7280) for body, Light Gray (#9CA3AF) for tertiary
     Borders: Subtle Gray (#E5E7EB)

     2. Typography

     - Heading 1 (32px, 700): Page titles
     - Heading 2 (24px, 600): Section headers
     - Heading 3 (20px, 600): Card titles
     - Body (16px, 400): Main content
     - Body Small (14px, 400): Secondary text
     - Caption (12px, 400): Tertiary information
     - Button (16px, 600): Action labels

     3. Spacing System

     - xs: 4px (borders, small gaps)
     - sm: 8px (inner padding, small gaps)
     - md: 16px (standard spacing)
     - lg: 24px (component spacing)
     - xl: 32px (section spacing)
     - 2xl: 48px (major section spacing)

     4. Component Styling

     Buttons

     - Primary: Solid background with subtle gradient, rounded corners (12px), shadow
     - Secondary: Outline with border, transparent background
     - Ghost: Minimal with only text, transparent background
     - Loading: Show activity indicator with text

     Cards

     - Elevated: White background, 12px border radius, subtle shadow (elevation 2)
     - Outlined: White background with 1px border, 12px border radius
     - Flat: Light gray background, 12px border radius

     Inputs

     - Floating labels or clear labels with focused states
     - Subtle borders with focus color change
     - Error states with red border and helper text

     Typography

     - Consistent font family (San Francisco for iOS, Inter for Android)
     - Proper line heights for readability
     - Color contrast ratios meeting WCAG AA standards

     Implementation Phases

     Phase 1: Foundation (Theme & Typography)

     1. Update theme/colors.ts with new sophisticated palette
     2. Update theme/typography.ts with refined font sizes and weights
     3. Update theme/spacing.ts to ensure consistent spacing
     4. Update theme/borderRadius.ts for consistent corner radii
     5. Update theme/elevation.ts for modern shadows

     Phase 2: Component Modernization

     1. Buttons: Update components/ui/Button.tsx with new styles, gradients, and animations
     2. Cards: Update components/ui/Card.tsx with modern styling and elevation
     3. Inputs: Update components/ui/Input.tsx with floating labels and better error handling
     4. Typography: Update components/themed-text.tsx to use new typography system
     5. EmptyState: Update components/ui/EmptyState.tsx with modern design

     Phase 3: Screen-Specific Updates

     1. Login/Register: Add subtle animations, better form layouts, modern inputs
     2. Home: Update service cards with better spacing, icons, and hover effects
     3. Orders: Improve order cards with better status indicators and spacing
     4. Custom Service: Modern form layout with better input grouping
     5. Messages: Update chat bubbles with better styling and read receipts
     6. Account: Modern profile card with better layout

     Phase 4: Navigation & Layout

     1. Update tab bar styling with modern icons and animations
     2. Add smooth transitions between screens
     3. Implement proper header hierarchy
     4. Add subtle animations for loading states

     Phase 5: Polish & Micro-interactions

     1. Add ripple effects on button presses
     2. Implement smooth transitions for form focus states
     3. Add subtle animations for empty states
     4. Implement pull-to-refresh animations
     5. Add loading skeleton screens

     Design Guidelines

     Color Usage

     - Primary color for main actions and key elements
     - Secondary/Accent for highlights and interactive states
     - Gray scale for backgrounds, borders, and text
     - Never use pure black (#000000) - use dark gray (#1F2937) instead

     Typography Hierarchy

     - Use Heading 1 for page titles only
     - Use Heading 2 for section headers
     - Use Heading 3 for card titles and important labels
     - Use Body for main content
     - Use Caption for secondary information
     - Always maintain proper contrast ratios

     Spacing Principles

     - Use consistent spacing units (multiples of 4px)
     - Group related elements with less space, separate unrelated with more space
     - Ensure touch targets are at least 44x44px
     - Use padding to create breathing room

     Component Design

     - Rounded corners (8px-12px) for modern feel
     - Subtle shadows for depth (elevation 1-4)
     - Consistent border radius across all components
     - Use gradients sparingly for visual interest

     Verification & Testing

     Visual Testing

     1. Test all screens in both light and dark mode (if applicable)
     2. Verify color contrast ratios meet WCAG AA standards
     3. Check spacing consistency across all components
     4. Validate typography hierarchy on all screens

     Functional Testing

     1. Test all button states (normal, loading, disabled)
     2. Verify form validation and error states
     3. Test tab navigation and transitions
     4. Verify accessibility features (screen reader support)
     5. Test on both iOS and Android devices

     Performance Testing

     1. Measure frame rates during animations
     2. Check bundle size impact of new styles
     3. Verify no layout thrashing from style recalculations
     4. Test on low-end devices for smoothness

     Risks & Mitigations

     Risk: Breaking existing functionality

     - Mitigation: Make incremental changes, test thoroughly after each change

     Risk: Inconsistent styling across screens

     - Mitigation: Use theme variables exclusively, create reusable component styles

     Risk: Performance impact from animations

     - Mitigation: Use native driver for animations, keep animations short (200-300ms)

     Risk: Accessibility issues

     - Mitigation: Test with screen readers, verify color contrast ratios
